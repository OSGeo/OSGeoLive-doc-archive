Guida rapida su TinyOWS¶

TinyOWS è un server Transactional Web Feature Service (WFS-T) ad alto rendimento, che è leggero e facile da installare, con un interfaccia CGI o FastCGI e utilizza PostGIS per archiviare i dati.

Questa guida rapida descrive come:

  • visualizzare un layer WFS in QGIS
  • editare un layer WFS usando WFS-T in QGIS

Visualizzare un layer WFS con QGIS¶

  1. Cliccare Desktop ‣ Desktop GIS ‣ Quantum GIS.
  2. Cliccare Layer ‣ Add PostGIS Layer....
    • Visualizzeremo prima il layer originale PostGIS
  3. Premere New.
  4. Inserite pgrouting come Connection name e Database
  5. Premere OK.
  6. Premere Connect.
  7. Selezionare ways nella lista delle tabelle
  8. Premere Add.
  9. Aumentare lo zoom
  10. Cliccare Layer ‣ Add WFS Layer....
    • Ora si aggiunge un layer WFS basato sulla stessa tabella
  11. Prenere New.
  12. Inserite tinyows come nome e http://localhost/cgi-bin/tinyows come URL
  13. Premere OK.
  14. Premere Connect.
  15. Selezionare ows:ways nella lista dei layer
  16. Controllare Only request features overlapping the current view extent.
  17. Premere OK.
    • Il layer WFS verrà visualizzato:

Editare dati con WFS-T¶

  1. Cliccare Layer ‣ Toggle Editing.
  2. Cliccare Edit ‣ Node Tool.
  3. Muovere una vertice (croce) in un’altra posizione
  4. Cliccare Layer ‣ Save edits.
  5. Nascondere il layer disattivando tows:ways nella lista dei layer
    • Dovreste vedere la via editate in un altro colore, poichè viene letto direttamente da PostGIS.

Ed ora?¶

Per imparare di più su TinyOWS, il punto d’inizione è la documentazione di TinyOWS sulla homepage MapServer.
