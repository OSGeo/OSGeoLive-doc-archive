Running in a Virtual Machine¶

Ce guide décrit une manière d’exécuter l’OSGeoLive dans une machine virtuelle VirtualBox. Pour les autres méthodes, suivez les liens depuis la section « Voir aussi » plus bas.

Avantages des machines virtuelles¶

  • La vitesse de réponse d’un VM est beaucoup plus rapide que sur le DVD, et généralement comparable à l’installation directement sur la machine hôte.
  • Tous les changements de configuration, les mises à jour logicielles et les fichiers enregistrés sont permanents, tout comme sur n’importe quel ordinateur « physique». ainsi:
  • Vous pouvez personnaliser et mettre à jour le système
  • Vous pouvez sauvegarder votre travail dans la VM
  • Vous pouvez installer des logiciels supplémentaires
  • Vous pouvez sauvegarder l’installation de votre machine virtuelle.

Prérequis système¶

  • RAM: 2 Go au moins. Le système Lubuntu fonctionne bien avec 1 Go de RAM, et vous aurez besoin de garder au moins la même quantité de mémoire pour votre système hôte. Ainsi, un total de 2 Go sera nécessaire pour un bon fonctionnement. Certaines applications, comme GeoServer, ont besoin de la machine virtuelle d’avoir au moins 2 Go de RAM. Donc, si possible, envisagez de fournir 2 Go ou plus pour votre machine virtuelle.
  • Espace disque dur de rechange : le fichier disque virtuel (vmdk) de live.osgeo.org décompressé occupe environ 10 Go. Vous voudrez également plus d’espace pour travailler sur la machine virtuelle plus confortablement. Donc 20 Go est une bonne recommandation.
  • Processeur: Tout CPU fera l’affaire, mais un processeur qui supporte la « Technologie de virtualisation » sera plus rapide. Vous pouvez vérifier le CPU de votre ordinateur ici

La plupart des machines produites au cours des 4-5 dernières années répondront à ces besoins.

Howto¶

Téléchargements

Téléchargez Virtual Box pour votre plate-forme et installez le logiciel. Sur Windows exécuter l’installateur, ou sur Ubuntu (et systèmes Linux similaires) faire ce qui suit:

apt-get install virtualbox-ose

En plus, téléchargez fichier de disque virtuel OSGeo (vmdk) depuis live.osgeo.org . Cliquez sur le bouton « Téléchargez la machine virtuelle ». Ensuite, décompresser le fichier téléchargé (en utilisant 7zip ).

Créer une machine virtuelle

Démarrez l’application VirtualBox et cliquez sur le bouton « New » pour créer une nouvelle VM, puis sur « Next ».

Entrez un nom tel que OSGeoLive, et choisissez Linux comme « Operating system », et Ubuntu comme « Version ».

Dans l’écran suivant, paramétrez la mémoire à 1024 Mo (ou plus si votre ordinateur hôte dispose de plus de 4 Go, comme dans la capture d’écran ci-dessous).

Passez à l’écran suivant et choisissez « Utilisez un fichier de disque dur virtuel existant ». Cliquez maintenant sur le bouton (icône d’un dossier) pour parcourir l’endroit où vous avez enregistré le fichier vmdk OSGeoLive. Sélectionnez ce fichier, appuyez sur Suivant et Créez.

Conseils et réglages de configuration

Une fois la VM créée, cliquez sur le bouton Paramètres. Dans la section « General », allez dans l’onglet « Advanced » et cliquez sur « Show at top of screen » pour avoir la barre d’outils Mini.

Accédez à la section « Affichage » et augmenter la mémoire vidéo à 32 ou 64 Mo. Dans certains systèmes d’exploitation, il peut être nécessaire d”« Activer l’accélération 3D » pour que l’affichage fonctionne correctement.

En plus, passez à la section « Répertoires partagés » et cliquez sur « Ajouter un dossier » (icône + verte à droite) pour trouver un répertoire sur votre ordinateur hôte que vous souhaitez partager dans la VM.

Vous pouvez choisir de rendre le dossier partagé en lecture seule et monté automatiquement. Une fois que le « chemin d’accès au dossier » et le « nom du dossier » sont définis, cliquez sur OK, puis à nouveau OK pour terminer et fermer la fenêtre de paramétrage.

Exécuter une machine virtuelle

Démarrez maintenant la VM en cliquant sur le bouton Démarrer (flèche verte).

You should install the virtualbox guest additions, this will allow you to use full-screen mode.

sudo apt-get install --yes virtualbox-guest-dkms virtualbox-guest-utils virtualbox-guest-x11

Lorsque vous installez l’iso dans la VM, le nom d’utilisateur doit être réglé sur « user »

L’utilisateur « user » n’est pas membre du groupe « users » dans la machine virtuelle OSGeoLive. Cela empêche certains outils d’écrire dans le répertoire de données et empêche certains logiciels de démarrer.

La solution est de réparer le vmdk avec la procédure unique suivante:

  1. Démarrer un terminal.
  2. Exécutez « sudo adduser user users ».
  3. Appliquez cette modification en commençant une nouvelle session de bureau : soit en redémarrant la machine virtuelle, soit vous vous déconnectez et vous reconnectez (nom d’utilisateur « user », mot de passe « user »).

Aussi une fois que le système OSGeo démarre, ajoutez-vous au groupe vboxsf afin que les dossiers partagés (définis ci-dessus) soient accessibles en cours d’exécution dans avec un terminal :

user@osgeolive:~$ sudo usermod -a -G vboxsf user

Dans l’exemple ci-dessus, nous avons défini un chemin d’accès dossier partagé sur le système hôte et l’avons nommé « GIS» dans les paramètres VM. Le dossier partagé s’affiche dans le système de fichiers sous /media/sf_GIS/. Pour monter ce dossier dans le répertoire d’accueil de l’utilisateur, par exemple, dans une exécution de terminal :

user@osgeolive:~$ mkdir GIS

user@osgeolive:~$ sudo mount -t vboxsf -o uid=user,rw GIS /home/user/GIS

Maintenant, les fichiers sur votre machine hôte dans le dossier choisi seront également disponibles dans la VM dans le dossier « GIS » du répertoire personnel de l’utilisateur.
