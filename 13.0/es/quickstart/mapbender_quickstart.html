Gía de Inicio Rápido de Mapbender¶

Mapbender es un marco de geoportal basado en web para publicar, registrar, ver, navegar, controlar y otorgar acceso seguro a servicios de infraestructura de datos espaciales.

Será necesario nada más que un navegador web estándar para este Inicio Rápido.

Esta Guía de Inicio Rápido describe cómo:

  • Iniciar Mapbender
  • crear una aplicación
  • Cargar un Web Map Service (WMS OGC)
  • configurar WMS
  • crear una aplicación individual
  • crear un usuario y un grupo y asignar aplicaciones a ellos

Se trata de cómo puede parecerse a una aplicación de Mapbender:

Instalación¶

Para la instalación de Mapbender echar un vistazo a https://doc.mapbender.org/en/installation.html.

Iniciar Mapbender¶

  1. Elejir Geospatial ‣ Browser Clients ‣ Mapbender del menú de inicio, o visite http://localhost/mapbender/app.php
  2. La aplicación tomará unos momentos para comenzar

Si tiene alguna dificultad ejecutando Mapbender, por favor compruebe si están ejecutando el servidor de web Apache y la base de datos PostgreSQL. O si usa SQLite como base de datos, compruebe si usted tiene acceso de escritura al archivo.

Inicie Mapbender en modo desarrollador app_dev.php¶

Symfony ofrece un modo de desarrollador con mucha información sobre su aplicación (registro, excepciones, consultas de base de datos, uso de memoria, tiempo y mucho más). Este modo sólo está disponible desde localhost.

  1. Iniciar el modo de desarrollador: http://localhost/mapbender/app_dev.php
  2. Echar un vistazo a la información que se ofrece en el modo de desarrollador.

Página de bienvenida¶

  1. La página de bienvenida enlista aplicaciones que son públicos y pueden ser utilizadas por todos los usuarios. Esas aplicaciones se enumeran con una captura de pantalla, un título y una descripción.
  2. Puede abrir una palicación haciendo click en el título, la captura de pantalla o el botón de inicio.
  3. Antes de que puede administrar con Mapbender tiene que iniciar sesión para obtener acceso a la administración.
  4. Haga click en Login en la parte superior derecha para abrir la página de inicio de sesión.
  5. Se puede acceder con el usuario que se generó en la instalación. Puede ser root y la contraseña root (Usuario y contraseña predeterminados que se obtienen después de la instalación de Mapbender en OSGeoLive. Por favor, cambie la contraseña de root si desea ejecutar un entorno productivo. Por favor no eliminar el usuario root.).

Después de un inicio de sesión exitoso, es dirigido a Mapbender administration.

Vista General de aplicación¶

Tras el inicio de sesión, usted es dirigido a Applications con una lista de aplicaciones a las que puede acceder.

La Vista General de la Aplicación proporciona la siguiente funcionalidad:

  • título y descripción
  • captura de pantalla para la aplicación (si se proporciona)
  • Enlace a la aplicación
  • botón para duplicar la aplicación
  • botón para editar la aplicación
  • botón para publicar/despublicar la aplicación
  • botón para eliminar la aplicación
  • botón para crear una nueva aplicación
  • con el campo de texto de filtro puede buscar aplicaciones

Crear una aplicación individual¶

Cree una nueva aplicación proporcionando información básica sobre su aplicación. Después, utilice el modo de edición para añadir elementos, capas y seguridad.

  1. elija Applications ‣ New Application
  2. defina un título y una descripción para la aplicación
  3. defina un título de URL que se utilizará en la URL para abrir la aplicación. Puede ser el mismo que el título
  4. suba un archivo de imagen como la captura de pantalla para ver la Vista General de la aplicación
  5. elija una plantilla para su aplicación
  6. Seleccione el botón Crear para crear la aplicación
  7. Haga clic en Edit-Button en la nueva aplicación para editar la aplicación
  8. la aplicación está configurada. Ahora necesita un WMS que se mostrará en su aplicación. Esto se describe en la sección Gestión de WMS.

Truco

Por favor, tome en cuenta que la configuración de estilo-, ícono- y diseño tiene hacerse en archivos-rama y css, el momento. Leer más sobre generación de plantillas en https://doc.mapbender.org/en/customization/templates.html.

Copiar o cambiar el nombre de una aplicación¶

También puede crear una nueva aplicación de copia de una aplicación existente. Se pueden copiar todas las aplicaciones con acceso de escritura. Proporciona un botón para duplicar la aplicación con todos los servicios. La nueva aplicación obtendrá un nombre derivado que puede ser cambiado en el paso siguiente.

Eliminar una aplicación¶

Puede eliminar una aplicación de la opción de menú Applications with the x-button.

Exportar / importar fuentes y aplicaciones¶

Puede exportar aplicaciones como JSON o YAML con Applications ‣ Export. Usted puede elegir una o más aplicaciones para exportar y también puede exportar las fuentes que se publican en las aplicaciones.

El exportfile contiene todas las definiciones de la aplicación/es (fuentes) y pueden ser importado a través de Applications ‣ Import en otra instalación de Mapbender3 o en la misma instalación.

Gestión de Fuentes de Datos¶

Mapbender puede manejar OGC WMS. La administración proporciona una interfaz de administración para las fuentes (de momento sólo WMS).

Vista General del Servicio de Repositorio¶

  1. Diríjase a Services y eche un vistazo al Servicio de Repositorio.
  2. Obtendrá una Vista General de las Fuentes que se cargan en su Mapbender.
  3. Tipo, Título, Descripción, se ofrece la primera información acerca de los servicios.
  4. Haga clic en el botón View-Button y obtendrá más información sobre la fuente.
  5. Con el botón Delete-Button puede eliminar la fuente del repositorio.

Cargar Servicios Mapas Web¶

Puede cargar Servicios de Mapas Web (WMS) de OGC para su aplicación.

Un WMS devuelve un archivo XML cuando el documento getCapabilities se solicita. Esta información se analiza por Mapbender y Mapbender obtiene toda la información necesaria sobre el servicio de este XML

Truco

Primero debe verificar el documento Capabilities en su navegador antes de intentar cargarlo con Mapbender

  1. Elegir el elemento en el menú Services ‣ Add Service o utilice el +-Button. Enlace a la URL getCapabilities de WMS en el campo de texto Serviceurl.
  2. Introduzca nombre de usuario y contraseña si su servicio requiere autenticación.
  3. Pulse Cargar para cargar el Servicio del repositorio.
  4. Después del registro exitoso del servicio de Mapbender se mostrará una Vista General sobre la información que fue proporcionada por el servicio.

Aquí está un ejemplo WMS:

WhereGroup OSM WMS https://osm-demo.wheregroup.com/service?SERVICE=WMS&VERSION=1.3.0&REQUEST=getCapabilities

Añadir Servicio a la Aplicación¶

Después de la carga exitosa de un WMS usted desea agregar su WMS a una aplicación.

  1. Elija Applications ‣ edit-Button ‣ Layers ‣ Edit-Button.
  2. Haga click en +-button para agregar un conjunto de capas, que representa una colección de capas (un nombre posible podría ser lo principal)
  3. Ahora puede añadir capas al conjunto de capas
  4. Haga click en +-button en su nuevo conjunto de capas para agregar un Servicio.
  5. Puede cambiar el orden de los Servicios en su aplicación arrastrando y soltando.

Configure su WMS¶

Puede configurar el WMS para su aplicación específica. Tal vez no desea proporcionar todas las capas o quieres cambiar el orden o títulos de la capa o desactivar la información de la característica o cambiar la escala máxima de una capa.

  1. Elija Applications ‣ edit-Button ‣ Layers ‣ edit-Button para configurar la Instancia de Servicio.
  2. Usted verá una tabla con las capas del Servicio.
  3. Puede cambiar el orden de las capas arrastrando y soltando

Configuración del servicio

  • formato - elija el formato para getMap-Requests
  • infoformat - elija el formato para getFeatureInfo-Requests
  • exceptionformat - elegir el formato para excepciones
  • opacidad - elija el porcentaje de opacidad
  • visible - predeterminado. El servicio debe ser visible
  • Fuante base - activada para los servicios que deben ser administrados por BaseSourceSwitcher
  • proxy - si activa el servicio se solicitará por Mapbender y no directamente
  • transparencia - El valor estándar está activado, desactivado si el servicio se solicita sin un fondo transparente (getMap-Request con TRANSPARENT = FALSE)
  • mosaico - usted puede solicitar un WMS en teselas, el predeterminado o es tesela (puede ser una buena opción si tu mapa es muy grande y el servicio WMS no es compatible con la anchura y altura)
  • Ordenamiento de capas - el ordenamiento de capas se puede cambiar en la aplicación
  • Factor BBOX - Este parámetro es válido para servicios WMS no inclinados. Puede especificar el tamaño de la imagen del mapa devuelto. Un valor mayor que 1 solicitará una imagen de mapa más grande. Predeterminado: 1.25, se puede establecer en 1.
  • búfer de teselas: este parámetro es válido para los servicios de teselas y especifica si se deben solicitar teselas adicionales. Si el usuario desplaza el mapa, estas teselas ya se han descargado y son visibles. Cuanto mayor sea el valor, más teselas son solicitadas. Predeterminado: 0

Configuración de capa

  • título - título de la capa de información de Servicio
  • activo (encendido/apagado) - activar/desactivar una capa para esta aplicación individual
  • seleccionar en - seleccionable en el explorador de geodatos
  • Seleccione permitir - la capa está activa cuando se inicia la aplicación
  • información sobre -la capa proporciona solicitudes de información de características, la información predeterminada activa la funcionalidad de información de características
  • info permitir - información de los atributos es posible para esta capa
  • escala mínima/ escala máxima - el rango de la escala en que se debe mostrar la capa, 0 significa que no hay limitación de escala
  • alternar - abrir la carpeta al inicio de la aplicación
  • reordenar - permite reordenar las capas arrastrando y soltando durante el uso de la aplicación
  • …-> abre un cuadro de diálogo con más información
  • nombre - nombre de capa de información de servicio (para getMap-Requests)
  • estilo - si un WMS ofrece que más de un estilo, usted puede elegir un estilo diferente al predeterminado

Agregar Elementos a la aplicación¶

Mapbender ofrece un conjunto de elementos. Puede Agregar los elementos a su aplicación. Tiene diferentes regiones (Barra de Herramientas, Sidepane, Contenido, Pie de Página) a las que usted puede agregar elementos.

  1. Elija Applications ‣ edit-Button ‣ Layers ‣ Button + para obtener una Vista General de los elementos que Mapbender3  proporciona
  2. Elija un elemento de la lista.
  3. Tenga en cuenta que tiene distintas áreas en su aplicación. Asegúrese de agregar el elemento a una región que haga sentido.
  4. Configurar el elemento. Aviso: Al seleccionar un elemento por ejemplo mapa puede ver que el elemento tiene un conjunto de atributos. Cada elemento ofrece atributos individuales para la configuración.
  5. Usted puede cambiar la posición del elemento a través de arrastrar y soltar
  6. Eche un vistazo a su aplicación. Puede abrirla desde Applications ‣ Applications Overview

Ahora usted debe tener una idea de lo fácil que es cambiar una aplicación de Mapbender sin cambios en el código.

Ejemplos de elementos que Mapbender ofrece:

  • Indicador de actividad
  • BaseSourceSwitcher o Conmutador de Fuente Base
  • Botón
  • Pantalla de Coordenadas
  • Utilidad de Coordenadas
  • Derechos de autor
  • Información de funciones
  • Posición GPS
  • HTML
  • Leyenda
  • Layertree - tabla de contenido
  • Mapa
  • Barra de herramientas de navegación (Zoombar)
  • Vista General
  • PrintClient
  • Redlining
  • Línea/Área de Regla
  • Selector de Escala
  • ScaleBar
  • SimpleSearch - Búsqueda a través de Solr
  • SearchRouter - Seach a través de SQL
  • Selector de SRS
  • Selector de sistema de referencia espacial (SRS Selector)
  • Cargador WMS
  • Editor WMC
  • Cargador WMC
  • Lista WMC

Encontrará información detallada sobre cada elemento de la “Documentación de elementos de Mapbender <https://doc.mapbender.org/en/functions.html>`_.

Inténtelo usted mismo¶

  • Añadir un elemento de mapa al contenido de su aplicación
  • añadir un Layertree al panel lateral de la aplicación
  • añadir una leyenda al panel lateral de la aplicación
  • añadir un elemento de impresión al contenido
  • añadir un botón que abra la impresión en la parte superior de la aplicación
  • Añadir la Barra de Herramientas de Navegación al contenido
  • añadir un mapa de información general al contenido
  • Añadir un Copyright y cambiar el texto de derechos de autor
  • añadir un selector de escalas y un selector SRS al pie de página

Gestión de usuario y grupo¶

Un acceso a Mapbender requiere autenticación. Sólo públicas aplicaciones se pueden utilizar por todo el mundo.

Un usuario tiene permisos tener acceso a uno o a un conjunto de aplicaciones y servicios.

Crear un usuario¶

  1. Para crear un usuario, vaya a New User o haga clic en +-Button.
  2. Elegir un nombre y una contraseña para tu usuario.
  3. Proporcionar una dirección de correo electrónico para el usuario.
  4. Guardar el nuevo usuario.
  5. Puede proporcionar más información sobre el usuario en la pestaña Profile.

Crear un grupo de¶

  1. Crear un grupo by New Group.
  2. Definir un nombre y una descripción para su grupo.
  3. Guarde su nuevo grupo.

Asignar usuarios al grupo¶

  1. Asignar un usuario a un grupo por Users –> Groups.
  2. Elegir uno o más usuarios que desea agregar al grupo en Users.
  3. Asignar un usuario a un grupo por Users ‣ Edit-Button‣ Groups.

Roles¶

Mapbender otorga diferentes derechos. Se refieren al Symfony ACL System http://symfony.com/doc/2.8/cookbook/security/acl_advanced.html#built-in-permission-map

  • vista - Si a alguien se le permite ver el objeto.
  • editar - si a alguien se le permite realizar cambios en el objeto.
  • eliminar - si a alguien se le permite eliminar el objeto.
  • operador - si a alguien se le permite llevar a cabo todas las acciones anteriores.
  • maestro - si alguien se permite llevar a cabo todas las acciones anteriores y además se le permite conceder cualquiera de los permisos anteriores a otros.
  • propietario - si alguien posee el objeto. Un propietario puede realizar cualquiera de las anteriores acciones y otorgar permisos maestro y propietario.
  1. Asignar roles a un usuario por Users ‣ Edit your User ‣ Security.

Asignar una Aplicación a un Usuario/Grupo¶

  1. Edite su aplicación por Application ‣ Edit-Button.
  2. Elija Security
  3. Configurar los permisos como vista, editar, eliminar, operador, dueño, propietario
  4. Asignar un usuario/grupo a la aplicación
  5. ¡Pruebe su configuración!
  6. Cierre sesión de Mapbender por Logout.
  7. Inicie sesión como el nuevo usuario

Cosas para probar¶

Aquí están algunos retos adicionales para que los pruebe:

  1. Intente cargar algunos WMS en su aplicación. Intente configurar su WMS.
  2. Intentar crear una aplicación individual.

¿Qué sigue?¶
